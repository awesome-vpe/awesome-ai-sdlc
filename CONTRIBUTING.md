# Contributing to Awesome AI-SDLC

Thank you for your interest in contributing! This list aims to be a high-quality, time-aware collection of resources about how GenAI is transforming the SDLC.

## Adding a Resource

### Format

```markdown
- 🟢 `[2026-02]` [Title of Resource](https://url) - One-line description of key finding or contribution.
```

### Evidence Ratings

Choose the most appropriate rating:

- **🟢 Empirical study** — Randomized controlled trial, large-scale observational data, peer-reviewed research
- **🟡 Industry report** — Survey-based research, vendor reports with methodology, analyst research (DORA, Gartner, Bain, etc.)
- **🔵 Practitioner insight** — Blog posts, conference talks, experience reports from recognized practitioners
- **🟠 Theoretical / opinion** — Thought leadership, predictions, frameworks without empirical validation yet

### Date Tags

Use `[YYYY-MM]` format based on publication date. If a resource is significantly updated, use the update date.

### Quality Bar

We prefer resources that offer:

- ✅ Specific findings, data points, or measurements
- ✅ Novel frameworks or mental models
- ✅ Reproducible methodology
- ✅ Actionable insights for engineering leaders

We generally avoid:

- ❌ Generic "AI will change everything" pieces
- ❌ Vendor marketing without substance
- ❌ Speculation without evidence or frameworks
- ❌ Duplicates of existing entries

### Placement

Place your resource in the most specific applicable section. If it spans multiple SDLC phases, place it in the "Cross-Cutting Concerns" section or the most relevant phase.

## Adding a Timeline Entry

If a resource represents a significant inflection point in the AI-SDLC transformation, add it to the [Timeline](#timeline) section as well.

## Proposing Structural Changes

For changes to the section structure, please open an issue first to discuss. The structure should reflect how the SDLC is actually transforming, not how we wish it would.

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Add your resource(s) following the format above
4. Submit a PR with a brief description of why the resource is notable

## Code of Conduct

Be respectful. This is a curated collection, not a link dump. Quality over quantity.
